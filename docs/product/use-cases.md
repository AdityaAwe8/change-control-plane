# Use Cases

## Startup Bootstrap Mode

An early-stage company connects a repository, creates dev, staging, and production environments, accepts default rollout and observability policies, and starts shipping with governed delivery and basic service scorecards on day one.

## Brownfield Advisory Mode

An existing company ingests changes, deployments, and topology from existing systems in read-only mode. The platform scores change risk, identifies missing ownership or observability coverage, and recommends rollout paths without blocking delivery.

## Policy-Governed Production Changes

A production change touching customer-facing services, IAM, and regulated data enters review. The platform computes risk, recommends canary rollout, requires additional approval, and records the policy rationale and audit trail.

## Incident-Aware Rollbacks

A deployment begins to degrade latency and checkout conversion. Runtime verification gates catch the regression, pause the rollout, and recommend rollback or forward-fix pathways tied to service criticality and past incident history.

## Enterprise Progressive Adoption

An enterprise keeps GitHub Actions, Argo CD, Datadog, PagerDuty, Slack, and Jira in place while using ChangeControlPlane as the governing layer for risk scoring, approvals, evidence, and controlled rollout recommendations.
