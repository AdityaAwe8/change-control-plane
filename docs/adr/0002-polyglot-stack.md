# ADR 0002: Go, Python, and TypeScript

## Status

Accepted

## Decision

Use:

- Go for APIs, workers, controllers, and CLI
- Python for advanced analytics and simulation work
- TypeScript for the web application and future client SDK layers

## Rationale

- Go gives operational simplicity and reliable concurrency for control-plane services
- Python remains the strongest fit for exploratory analytics and model-heavy future work
- TypeScript is the best fit for a rich operational web application and typed client integration
